Profe, esta fue la solucin para leer los archivos. No s si la intencin sea leer usando la funcin read.delim() por que la forma de los datos no permite por default leer la base de datos. Es necesario complementarla con algo ms. En mi caso use la libreria tidyr que hace parte de tidyverse.

La solucin que implemente, fue la siguiente:

read.delim("jan17Weather.csv") %>% tidyr::separate("date.PRCP.SNOW.TMAX.TMIN", c("date", "PRCP", "SNOW", "TMAX", "TMIN"), sep = "\t") %>% mutate(date = as.Date(date)) %>% mutate_if(is.character, as.numeric) %>% head()

Esto de acuerdo a, que al leer el archivo nos va a retornar un dataframe con solo una columna. De esta forma:

# Not run
read.delim("jan17Weather.csv")

Al sacar los nombres del data frame, evidenciamos que nicamente tenemos una columna.

read.delim("jan17Weather.csv") %>% names()

[1] "date.PRCP.SNOW.TMAX.TMIN"

Donde, la primera y ltima columna tienen un ". Esto puede ser contra intuitivo y sucede porque los datos del archivo .csv tienen la siguiente estructura

Anotacin uno

con unas " al inicio y al final entendiendo la estructura de datos como si fuese una sola columna de tipo string.

O si aplicamos el atributo quote de la funcin read.delim() nos retornara la siguiente tabla, la cual supongo es la que usted tiene:

# Not run
read.delim("jan17Weather.csv", quote = "")

Resultando esta forma de lectura contra intuitiva, porque puede tender a pensarse que por tener " el atributo quote de la funcin read.delim() leera los valores sin problemas, pero este atributo nicamente funciona cuando cada una de las columnas esta rodeada de " al inicio y al final.

Para solucionar esto, existen algunas alternativas. Podemos, reemplazar directamente el archivo .csv quitandole las " y despus volviendo a leer el archivo, esto lo solucionara.

Anotacin dos

# Not tun
read.delim("jan17Weather - ejemplo.csv")

La base de datos jan17Weather - ejemplo.csv contiene la modificacin quitando las " del archivo original como muestra la Anotacin dos.

sin embargo, hacer esto puede ser complejo si estamos conectados a una base de datos o si el archivo o base de datos contiene un alto nivel de datos o si no se quiere modificar la base de datos inicial. Puede usarse la combinacin de los siguientes comandos, los cuales use para leer el archivo .csv, Donde:

# Not run
read.delim("jan17Weather.csv") %>% tidyr::separate("date.PRCP.SNOW.TMAX.TMIN", c("date", "PRCP", "SNOW", "TMAX", "TMIN"), sep = "\t")

Uniendo el mtodo read.delim() en un pipeline %>% con la funcin separate() de la libreara tidyr, ac la documentacin. Separando una columna con multiples columnas, es decir, separaremos la columna que nos lee read.delim("jan17Weather.csv") indicandole:

  1. El nombre de la columna actual: "date.PRCP.SNOW.TMAX.TMIN"
  2. Un vector con los nombres de la columna a dividir: c("date", "PRCP", "SNOW", "TMAX", "TMIN")
  3. La expresin regular o REGEX, que separar la columna en cada una de las columnas: SEP = " \t"

Despus de separa los valores en columna, vemos que R an tiene los valores de columnas como caracteres, por lo tanto, implementamos un mutate() para modificar el tipo de los vectores dentro del data frame. De la siguiente manera:

Donde, dentro del mismo pipeline indicamos: mutate(date = as.Date(date)) para modificar la columna date a tipo <date> y luego con el comando: mutate_if(is.character, as.numeric) indicamos que nos modifique todas las columnas que sean tipo caracter, en este caso las columnas faltantes PRCP, SNOW, TMAX, TMIN y que las transforme a tipo <dbl>. Es decir:

  1. Todas las columans son caracteres.
  2. El comando: mutate(date = as.Date(date)) modifica las columna date a tipo fecha.
  3. El comando: mutate_if() necesita una condicin, en nuestro caso es is.character() metodo que devuelve valores booleanos. Y necesita un metodo para modificar los valores que cumplan la condicin, en nuestro caso as.numeric() mtodo que transforma valores a numeros (incluidos tipo double o <dbl>).

Es necesario denotar que los argumentos ingresados a mutate_if son mtodos sin prentesis.
